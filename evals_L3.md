# 5-Dimension QA Framework for Weekly L3 Individual Productivity Issues

### Dimensions:

1. **Production Validation**  
   - Evidence that the shortcut/template works in a real context.  
   - Examples include case studies, user reviews, or personal deployments with measurable results.  

2. **Reusability**  
   - Classify the shortcut/template as one of:  
     - Personal hack  
     - Team-shareable  
     - Cross-org shareable  

3. **Time-Value Ratio**  
   - Clearly state setup effort vs. weekly time savings.  
   - Required to have low setup effort and high savings for a pass.  

4. **Security & Compliance**  
   - Disclose all risks associated with the shortcut/template.  
   - Include data handling implications and regulatory context for fintech/e-commerce items.  

5. **Role Specificity**  
   - Ensure the template is tailored specifically to a role's workflow and not just generic productivity advice.  

### Pass/Fail Criteria:

- **Pass**  
  - Clear evidence provided for each dimension.  
  - Examples include validated experiences and appropriate classification of tools.  
- **Fail**  
  - Lack of evidence or generic content.  
  - Hidden setup efforts, unvalidated shortcuts, or missing security notes.

### Anti-Patterns:
- Unvalidated shortcuts that may lead to negative consequences.
- Generic content that doesn't apply to specific roles or workflows.
- Hidden setup effort leading to unexpected time investments.
- Lack of security notes resulting in potential compliance issues.

### 20-Point QA Checklist:
1. Evidence of production validation exists.  
2. Clear classification of reusability.  
3. Realistic time-value ratio provided.  
4. Risks and compliance issues outlined.  
5. Role specificity demonstrated.  
6. Evidence validated by external review.  
7. User feedback resembles positive results.  
... (add additional checks as needed)
